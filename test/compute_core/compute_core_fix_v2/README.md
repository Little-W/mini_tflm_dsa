# compute_core control-timing fix

This version fixes the compute-core/ps-buffer timing according to the required external protocol:

1. IA input is already diagonalized/stair-stepped by the testbench.
2. `ia_row_valid`, `ia_is_init_data`, and `ia_calc_done` follow the last input row.
   For `SIZE=4`, they are asserted during input steps 3, 4, 5, and 6.
3. The first tile asserts `ia_is_init_data` together with `ia_row_valid`.
4. The final tile asserts `ia_calc_done` together with `ia_row_valid`.
5. `send_ia_trigger` is pulsed one cycle before the next stair-stepped IA stream.
6. `bias_sleep=0` only during the first tile; otherwise it is `1`.
7. The external TB still sends `ia_l1_switch` after each tile input completes.

## Important internal fix

The external `ia_row_valid` window is only `SIZE` cycles, but the partial-sum stream observed by `ps_buffer` is a complete diagonal stream of `2*SIZE-1` cycles. Therefore `compute_core_test` now generates an internal full partial-sum window:

```systemverilog
ps_stream_valid = 2*SIZE-1 cycles
```

This internal window starts on the rising edge of external `ia_row_valid`.

For the final tile, the internal `ps_calc_done` is also extended to the same `2*SIZE-1` cycle window, so `de_diagonalizer` receives the complete diagonal stream instead of only the last `SIZE` cycles.

## ps_buffer change

`ps_buffer` now stores/replays the complete diagonalized vector stream. All lanes use the same FIFO control signals. The FIFO path does not generate `acc_data_valid` or `tile_calc_over`; both are still exclusively generated by `de_diagonalizer`.

The internal `ps_l1_switch` is generated on the last cycle of the internal partial-sum window so that the FIFO RAM frame length includes the whole `2*SIZE-1` diagonal stream.

## TB checks

The TB checks both paths:

- For intermediate tiles, FIFO replay must match the previous accumulated partial sums in diagonal/stair-step order.
- For the final tile, `acc_data_valid` must be asserted with each de-diagonalized output row.
- `tile_calc_over` must be asserted for exactly one cycle with the final valid output row.

Waveform dump is kept:

```systemverilog
$dumpfile("tb_compute_core.vcd");
$dumpvars(0, tb_compute_core);
```
